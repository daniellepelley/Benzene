**2026-07-14 correction:** This document's "Test Helpers Extraction (100% Complete)" claim
below was not actually accurate as of 2026-07-11 — it also contradicted this same document's
own "Generated Documentation" section further down, which listed
`test-helpers-refactoring-plan.md` as "❌ PENDING - needs execution". A direct code audit today
found 3 real remaining leaks the earlier pass missed: `Benzene.Aws.Lambda.ApiGateway/
MessageBuilderExtensions.cs`, and `BenzeneTestHostExtensions.cs` in both
`Benzene.Aws.Lambda.Core` and `Benzene.Azure.Function.Core` (the latter two meant those
production packages carried a `ProjectReference` to the test-only `Benzene.Testing` package).
All 3 have now been fixed (moved into new/existing `.TestHelpers` sibling packages,
`Benzene.Testing` references removed from production `.csproj` files) — details were recorded in
`work/test-helpers-refactoring-plan.md`, since deleted as resolved (see git history). As of
2026-07-14, "no test code in
production packages" is now genuinely verified true via a repo-wide sweep, not just claimed.
No other section of this document was re-verified in this pass — the "Step 6: Publish"
readiness assessment below should be treated with the same skepticism until independently
re-checked; `version.txt` is still `0.0.2`, so no publish action has actually been taken.

**2026-07-14 addendum (later same day):** `Benzene.Http`'s CORS support (`CorsSettings`,
`CorsMiddleware<TContext>`) gained new public members after Step 1's API Surface Review was
completed — `CorsSettings.ExposedHeaders`/`AllowCredentials`/`MaxAgeSeconds`, added to bring CORS to
full spec parity with `Microsoft.AspNetCore.Cors`. Purely additive (no existing signature changed),
so it doesn't invalidate the review, but worth a quick look before `Benzene.Http`'s API is
considered fully locked for 1.0 semver purposes, since this document lists it as a "Ship at 1.0.0"
package (see Release Strategy below).

- ⚠️ **Discovered:** `BenzeneServiceContainerExtensions.AddScoped<T>(T implementation)` is

  with the identical signature, so C# always resolves to that (unconditional) method instead
  of the "Try" extension. The extension's XML doc promises dedup semantics it can never
  deliver. Not fixed (would be a breaking/behavioral change) — flagging for maintainer decision.

### Conservative Approach (RECOMMENDED)

- Benzene.Abstractions
- Benzene.Abstractions.Middleware
- Benzene.Core
- Benzene.Core.Middleware

- Reduces scope and risk for 1.0
